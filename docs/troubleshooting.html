üîß Troubleshooting

Common issues, solutions, and helpful tips

‚ùå "API key not found"

Make sure you've set the API key in .env or pass it directly:

from flip import Flip, FlipConfig

# Option 1: Set in .env file
# OPENAI_API_KEY=your-key-here

# Option 2: Pass directly in config
config = FlipConfig(openai_api_key="your-key-here")
flip = Flip(directory="./docs", config=config)

‚ùå "No documents found"

Check the following:

  • Verify your directory exists and path is correct
  • Check file extensions are supported
  • Verify file permissions
# Check supported extensions
from flip.document_processing.loader import DocumentLoader

extensions = DocumentLoader.get_supported_extensions()
print(f"Supported: {extensions}")

# Supported formats:
# Documents: PDF, DOCX, TXT, MD
# Data: JSON, CSV
# Web: HTML, HTM
# Code: Python, JavaScript, TypeScript, Java, C++, Go, Rust, Ruby

‚ùå Import Errors

If you encounter import errors:

# Reinstall dependencies
pip install -r requirements.txt

# Or install in editable mode
pip install -e .

# Verify installation
python -c "from flip import Flip; print('Success!')"

üêå Slow Performance

Try these optimizations:

  • Enable Caching: Speeds up repeated queries
  • Adjust Chunk Size: Larger chunks = fewer embeddings
  • Disable Re-ranking: Faster but less accurate
  • Use Local Models: Ollama + Sentence Transformers
  • Batch Processing: Increase batch_size
from flip import FlipConfig

# Optimized for speed
config = FlipConfig(
    chunk_size=1024,  # Larger chunks
    use_reranking=False,  # Disable for speed
    enable_cache=True,  # Enable caching
    batch_size=64  # Larger batches
)

üìâ Poor Results Quality

Improve accuracy with these settings:

  • Enable Hybrid Search: Better retrieval
  • Enable Re-ranking: More accurate results
  • Try Different Chunking: Semantic often works best
  • Adjust top_k: Retrieve more documents
  • Better LLM Model: Use GPT-4 instead of GPT-3.5
from flip import FlipConfig

# Optimized for accuracy
config = FlipConfig(
    llm_model="gpt-4-turbo-preview",
    chunking_strategy="semantic",
    chunk_size=512,
    retrieval_top_k=10,  # More context
    use_hybrid_search=True,
    use_reranking=True
)

üíæ Memory Issues

Reduce memory usage:

  • Process documents in smaller batches
  • Use smaller embedding models
  • Reduce cache size
  • Clear cache periodically
from flip import FlipConfig

config = FlipConfig(
    batch_size=16,  # Smaller batches
    embedding_model="all-MiniLM-L6-v2"  # Smaller model
)

# Clear cache when needed
flip.clear()

üí° Tips & Best Practices

  • Start Small: Test with a few documents first
  • Check Stats: Use flip.get_stats() to see indexing info
  • Try Different Strategies: Experiment with chunking strategies
  • Use Local Models: Try Ollama + Sentence Transformers for free usage
  • Check Citations: Always review response.citations for sources
  • Monitor Performance: Use flip.get_monitoring_stats()
  • Enable Logging: Helps debug issues
  • Profile Queries: Use profile_query() to find bottlenecks
