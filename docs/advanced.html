Logging Setup

from flip.utils.logger import setup_logging

# Setup logging
logger = setup_logging(
    level="INFO",
    log_file="./flip_logs/flip.log",
    console=True
)

# Use logger
logger.info("Flip SDK initialized")
logger.debug("Debug information")
logger.warning("Warning message")
logger.error("Error occurred")

FlipDebugger

from flip.utils.debug import FlipDebugger

# Create debugger
debugger = FlipDebugger(
    enabled=True,
    output_dir="./flip_debug"
)

# Trace events
debugger.trace("retriever", "search", {"query": "test", "top_k": 5})

# Time operations
debugger.start_timer("embedding_generation")
# ... do work ...
debugger.stop_timer("embedding_generation")

# Get timing
duration = debugger.get_timing("embedding_generation")
print(f"Took {duration:.4f}s")

# Print all timings
debugger.print_timings()

# Dump to files
debugger.dump_traces("traces.json")
debugger.dump_timings("timings.json")

from flip import Flip

    query="What is AI?",
    iterations=10
)

# Output shows:

Query Enhancement

Flip includes advanced query enhancement features:

  • Query Classification: Factual, Analytical, Creative, Conversational
  • Query Rewriting: Makes queries more specific
  • Query Decomposition: Breaks complex queries into sub-queries
  • HyDE: Hypothetical Document Embeddings for better retrieval

Custom Providers

from flip.generation.factory import LLMFactory
from flip.generation.base import BaseLLM

class MyCustomLLM(BaseLLM):
    # Implement required methods
    pass

LLMFactory.register_provider("mycustom", MyCustomLLM)

# Use it
flip = Flip(directory="./docs", llm_provider="mycustom")

Performance Tuning

from flip import FlipConfig

config = FlipConfig(
    # Chunking
    chunking_strategy="semantic",  # or "token", "sentence", "recursive"
    chunk_size=512,
    chunk_overlap=50,
    
    # Retrieval
    retrieval_top_k=5,
    use_hybrid_search=True,  # Better accuracy
    use_reranking=True,      # Even better accuracy
    
    # Caching
    enable_cache=True,
    cache_dir="./flip_cache",
    
    # Performance
    batch_size=32,
    show_progress=True
)

üí° Best Practices

  • Start Small: Test with a few documents first
  • Use Appropriate Log Levels: DEBUG for development, INFO for production
  • Enable Debugging for Development: Disable in production
  • Profile Before Optimizing: Find bottlenecks first
  • Inspect Components: Verify embeddings and chunks quality
  • Use Caching: Especially for production deployments
  • Monitor Performance: Track query times and accuracy
